手机保活建议

为防止手机息屏后系统自动清理后台进程导致数据中断，请在手机上进行以下设置：

  • 在手机设置中，将「心率中转助手」的电池优化设置为「不限制/无限制」。
  • 允许该 App 的「自启动」与「后台运行」权限。
  • 在手机多任务界面中，将「心率中转助手」卡片加锁/锁定。

💡 算法与数据源说明：
1. ⌚ 硬件级真实数据：连接支持心跳间期 (R-R 间期) 广播的专业传感器（如心率带）时，系统采用标准 RMSSD 算法进行医学级的自主神经系统健康分析。
2. 🌐 网络模拟估算：连接普通手环/手表（仅广播每秒平均 BPM）时，系统利用相邻心率波动和生理特征补偿系数进行估算，能有效捕捉并展示疲劳与压力的发展走向。

机器人预警推送设置

当心率超过安全阈值时，自动向指定的机器人发送即时警报通知。

推送预警阈值 (BPM) 当实时心率达到或超过该数值时触发推送，并与主页警报同步。
推送冷却时间 (分钟) 防止心率持续超标时频繁轰炸手机。在此时间内仅会发送一次预警。
微信推送 (PushPlus) 通过微信公众号接收心率异常提醒。请在 pushplus.plus 获取 Token。
Telegram 预警机器人 直接向您的 TG 账号或频道发送警报。
钉钉群自定义机器人 向钉钉群内推送心率警告。填入 Webhook 链接中的 access_token 值。
飞书群自定义机器人 向飞书群内推送警告。填入 Webhook 链接末尾的 Token / UUID 串。

服务端参数配置

实时调整后端运行参数。注：敏感操作需管理员权限。

云端部署模式 (Cloud Mode) 开启后仅监听 127.0.0.1 并关闭 UDP 局域网广播。建议生产环境开启并搭配反向代理使用。
Web 服务端口 (Server Port) 后台服务运行绑定的 Web 端口（默认 8765）。修改后需要手动重启后端生效。
安全 API Token (用于手机 App / 悬浮窗连接校验) 客户端握手报头中的 Bearer 凭证，请妥善保管。
修改网页端登录密码 请输入当前密码以验证身份，并设置符合安全要求的长密码。

数据维护与清理 (危险操作)

清空或重置系统保存的数据，操作生效后不可撤销。

清空所有历史心率记录 删除本地数据库中保存的全部心率记录，释放空间并重置统计指标。
